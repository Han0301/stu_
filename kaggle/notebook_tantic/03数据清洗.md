### 一、 缺失值处理 (Missing Value Imputation)

#### 1. Age（年龄）：连续型变量的填补
**定义**：由于部分乘客未登记或未找到记录，导致 `Age` 列存在空值（NaN）。直接删除会损失大量数据。

**痛点**：`Age` 分布不是正态分布（有婴儿、老人、壮年），用均值（Mean）填补会拉低方差，用中位数（Median）更稳健。

**Pandas 操作**：
```python
import pandas as pd
import numpy as np

# 假设 train 是你的 DataFrame
# 查看缺失情况
print("缺失值统计:\n", train.isnull().sum())

# 使用中位数填补 Age
# 为什么用 median？因为 Age 中有极端值（80岁老人），中位数比均值更能代表“典型人群”
median_age = train['Age'].median()
train['Age'] = train['Age'].fillna(median_age)

# 验证
print(f"填补后的 Age 中位数: {train['Age'].median()}")
```

**生动解释**：
想象你在排队领救生艇座位，队伍里突然有几个人的年龄没写。如果你随便填个“30岁”（均值），可能会把几个本该是“儿童”的位置给占了。**中位数**就像是队伍最中间的那个人的年龄，用它来填，能保证队伍的整体结构不被打乱。

---

#### 2. Embarked（登船港口）：类别型变量的填补
**定义**：极少数乘客的登船地点缺失。

**痛点**：这是分类变量（C=Cherbourg, Q=Queenstown, S=Southampton）。

**Pandas 操作**：
```python
# 查看哪个港口的人最多
print(train['Embarked'].value_counts())
# 输出通常是 S > C > Q

# 用众数（Mode）填补，即人最多的港口
mode_embarked = train['Embarked'].mode()[0]
train['Embarked'].fillna(mode_embarked, inplace=True)
```

**生动解释**：
这就好比你要去机场接一个忘了说从哪个门出来的朋友。你看了一眼大屏幕，发现**S门（南安普顿）**出来的人最多。那你站在S门等，猜对的概率最大。这就是**众数填补**的逻辑。

---

#### 3. Cabin（船舱号）：果断放弃或简化
**定义**：大量缺失（超过77%）。

**Pandas 操作**：
```python
# 策略：提取首字母作为 Deck（甲板层），缺失的标记为 'Unknown'
train['Deck'] = train['Cabin'].apply(lambda x: x[0] if pd.notnull(x) else 'U')
```

**生动解释**：
泰坦尼克号的船舱号像酒店房间号（如 C85, B28）。**C** 代表 C 层甲板。大多数人都没记录，与其瞎猜房间号，不如只看他在哪一层。没有记录的，我们就标记为 **U（Unknown）**。这样就把复杂的字符串变成了简单的分类特征。

### 二、 特征工程 (Feature Engineering)

#### 1. 家庭规模 (Family Size)
**定义**：利用 `SibSp`（兄弟姐妹配偶数）和 `Parch`（父母子女数）构造新特征。

**Pandas 操作**：
```python
# 家庭总人数 = 本人 + 兄弟姐妹 + 父母子女
train['Family_Size'] = 1 + train['SibSp'] + train['Parch']

# 对家庭规模进行分箱 (Binning)
# 0: 独自一人 (Alone)
# 1-3: 小家庭 (Small)
# 4+: 大家庭 (Large)
train['Family_Type'] = pd.cut(
    train['Family_Size'], 
    bins=[0, 1, 4, np.inf], 
    labels=['Alone', 'Small', 'Large']
)
```

**生动解释**：
在灾难面前，“一个人”和“一家人”的行为模式完全不同。
*   **独行侠（Alone）**：可能行动最快，但也最容易被遗忘。
*   **小家庭（Small）**：夫妻带一个孩子，通常会互相照应，生存率往往最高。
*   **大家庭（Large）**：人太多，集合慢，容易走散，反而生存率低。
通过 `pd.cut` 我们把数字变成了**行为模式标签**。

---

#### 2. 票价修正 (Fare Correction)
**定义**：测试集中有时会出现票价为 0 或缺失的情况。

**Pandas 操作**：
```python
# 查看票价分布，发现有很多极低值
print(train['Fare'].describe())

# 将 Fare 进行对数变换 (Log Transformation) 以消除长尾分布
# 为什么要 Log？因为 500美元和 600美元的差距，与 5美元和 105美元的差距意义不同
train['Fare_Log'] = np.log1p(train['Fare'])
```

**生动解释**：
如果不做处理，模型会觉得那个花 500 英镑买头等舱票的富豪比只花 5 英镑的穷人重要 100 倍。但实际上，**相对差距**更重要。Log 变换就像把显微镜调了一下焦距，让高低票价在模型眼中的“距离”变得更合理。

### 三、 数据类型转换 (Encoding)

#### 1. 独热编码 (One-Hot Encoding)
**定义**：将分类变量（如 Sex, Embarked）转换为机器学习算法能读懂的 0/1 矩阵。

**Pandas 操作**：
```python
# 将性别转换为 0 和 1 (Label Encoding 的一种特例)
train['Sex'] = train['Sex'].map({'male': 0, 'female': 1})

# 对 Embarked 进行 One-Hot Encoding
train = pd.get_dummies(train, columns=['Embarked'], prefix='Embarked')

# 查看结果
print(train[['Embarked_C', 'Embarked_Q', 'Embarked_S']].head())
```

**生动解释**：
机器看不懂 “Male” 或 “Female”。我们要给它戴上有色眼镜：
*   如果是男性，给他戴上 `[1, 0]` 的眼镜。
*   如果是女性，给他戴上 `[0, 1]` 的眼镜。
`get_dummies` 会自动帮你把 `Embarked` 这一列拆成三列（C, Q, S），每一列只有 0 或 1。

### 四、 实战技巧：防止“数据泄露”

这是一个非常严谨且重要的点。

**定义**：在数据清洗时，必须分别对 Train 和 Test 进行处理，或者只在 Train 上计算统计量。

**错误示范**：
```python
# 错误！用到了未来的信息
all_data = pd.concat([train, test])
all_data['Age'] = all_data['Age'].fillna(all_data['Age'].median())
```

**正确示范**：
```python
# 正确！只在训练集上计算中位数
age_median = train['Age'].median()
train['Age'].fillna(age_median, inplace=True)
test['Age'].fillna(age_median, inplace=True) # 用训练集的统计量填测试集
```

**生动解释**：
这就像考试。训练集是你的**复习资料**，测试集是**考卷**。你不能在做考卷的时候偷看复习资料的答案（全局均值）。你必须只用复习资料里算出来的公式去答题。

### 五、 总结：你的清洗流水线

| 步骤 | 操作 | 核心函数 | 目的 |
| :--- | :--- | :--- | :--- |
| **1. 查漏** | 检查缺失值 | `isnull().sum()` | 了解战场情况 |
| **2. 补缺** | 数值填补 | `fillna()`, `median()` | 修复数据漏洞 |
| **3. 造数** | 构造特征 | `+`, `apply()`, `cut()` | 创造新的生存线索 |
| **4. 变身** | 编码转换 | `get_dummies()`, `map()` | 翻译成机器语言 |
| **5. 对齐** | 列名一致 | `align()` 或手动补齐 | 确保模型能运行 |

通过以上步骤，原本杂乱无章的原始数据（Raw Data）就变成了干净的特征矩阵（Feature Matrix）。这时候扔进 RandomForest 或 XGBoost，分数通常会稳定在 **0.78 - 0.80** 左右。
