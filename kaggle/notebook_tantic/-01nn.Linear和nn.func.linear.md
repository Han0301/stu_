# torch.nn.Linear vs torch.nn.functional.linear 核心区别

**核心结论**：`nn.Linear` 是带参数管理的 Module 类，`F.linear` 是无状态的纯函数，`nn.Linear` 的 forward 本质是调用 `F.linear` 实现计算。

---

### 一、本质类型与参数管理（最关键）

| 特性 | torch.nn.Linear | torch.nn.functional.linear |
| :--- | :--- | :--- |
| **类型** | 继承 `nn.Module` 的类，是可训练层组件 | 纯函数，无状态，仅执行计算 |
| **参数管理** | 自动创建并注册 `weight` 和 `bias` 参数，可通过 `model.parameters()` 获取，支持优化器自动更新 | 需手动传入 `weight` 和 `bias` 张量，不自动注册，需手动管理梯度与更新 |
| **参数初始化** | 自动初始化（Kaiming 均匀分布），无需手动操作 | 需手动初始化参数张量，无默认初始化机制 |

---

### 二、用法与接口差异

```python
# nn.Linear用法：定义层时指定维度，参数自动生成
import torch.nn as nn
linear_layer = nn.Linear(in_features=10, out_features=5)  # 自动创建weight(5,10)和bias(5)
output = linear_layer(input_tensor)  # 直接传入输入，内部调用F.linear

# F.linear用法：需手动准备并传入参数
import torch.nn.functional as F
weight = torch.randn(5, 10)  # 手动创建权重
bias = torch.randn(5)        # 手动创建偏置
output = F.linear(input_tensor, weight, bias)  # 显式传入所有参数
```

---

### 三、计算逻辑与功能一致性

- 两者核心计算完全相同：**y = xAᵀ + b**（A 为 weight 矩阵）
- 输入输出形状规则一致：
  - 输入：`(*, in_features)`（* 表示任意维度）
  - 输出：`(*, out_features)`
- `nn.Linear` 的 forward 方法源码：`return F.linear(input, self.weight, self.bias)`

---

### 四、适用场景对比

| 场景 | 推荐选择 | 原因 |
| :--- | :--- | :--- |
| 构建标准神经网络层 | nn.Linear | 自动管理参数，集成到模型中更便捷，支持 `model.parameters()` 统一优化 |
| 参数共享 / 动态参数 | F.linear | 可手动传入同一参数到多个层，或动态生成 / 修改参数 |
| 自定义层实现 | 两者结合 | 自定义 Module 中用 F.linear 实现计算，手动管理特殊参数 |
| 性能优化 | 无显著差异 | 底层计算逻辑一致，性能取决于参数管理方式 |

---

### 五、关键注意事项

1. **参数注册**：`nn.Linear` 的参数自动加入模型参数列表，`F.linear` 的参数需手动调用 `nn.Parameter()` 包装才能被优化器更新
2. **设备迁移**：`nn.Linear` 可通过 `model.to(device)` 统一迁移所有参数，`F.linear` 需手动将输入、`weight`、`bias` 都移到同一设备
3. **序列化**：`nn.Linear` 可通过 `state_dict()` 保存参数，`F.linear` 需手动保存 / 加载参数张量

---

### 总结

**选择原则**：有可学习参数且需自动管理时用 `nn.Linear`；需灵活控制参数或无学习参数时用 `F.linear`。两者本质计算一致，差异集中在参数管理与使用便捷性上。
