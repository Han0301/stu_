在Kaggle经典的泰坦尼克号（Titanic）数据集中，**千万不要把“Name”仅仅当成一串无意义的文本符号**。

在1912年的泰坦尼克号上，逃生遵循着严格的潜规则：“妇孺优先”以及“阶级特权”。而名字（Name）字段中隐藏的**“头衔（Title）”**，恰好完美融合了**性别、年龄段**和**社会地位**这三个决定生死的关键要素。

因此，对Name建模的核心逻辑是：**提取头衔（Title） -> 映射社会地位 -> 转换为分类变量**。

以下是实战中最有效、得分最高的特征工程步骤：

### 第一步：提取“头衔 (Title)”
泰坦尼克号数据集中的名字格式高度统一，通常为：`姓氏, 头衔. 名字`。我们可以使用正则表达式轻松把头衔提取出来。

```python
import pandas as pd

# 假设 train 和 test 是你的 DataFrame
combined_data = [train, test]

# 使用正则表达式提取 Mr, Mrs, Miss, Master, Dr 等头衔
for dataset in combined_data:
    dataset['Title'] = dataset['Name'].str.extract(' ([A-Za-z]+)\.', expand=False)

# 查看提取出的所有独特头衔
print(train['Title'].value_counts())
```
运行后你会发现，除了常见的 `Mr`, `Mrs`, `Miss`, `Master`，还有一些特殊的头衔，比如 `Dr`（博士/医生）、`Rev`（牧师）、`Col`（上校）、`Countess`（女伯爵）等。

### 第二步：头衔映射与社会地位分组（核心操作）
不同的头衔代表了不同的社会阶层和群体。为了降低特征的维度并提高模型的预测能力，我们需要把这些零散的头衔进行**业务合并**：

*   **Officer（高级职员/专业人士）**：如 `Capt`（上尉）、`Col`（上校）、`Dr`（博士）、`Major`（少校）、`Rev`（牧师）。这些男性通常有较高的社会地位和领导力，可能更倾向于维持秩序或最后逃生。
*   **Royalty（贵族）**：如 `Don`、`Sir`、`Lady`、`the Countess`（女伯爵）、`Jonkheer`（荷兰贵族）。这类人极度稀少，但享有极高的特权。
*   **Miss / Mrs / Mr / Master**：普通大众。注意，`Miss` 和 `Mrs` 有时会被笼统地记为 `Ms` 或 `Mlle` (法语小姐)，需要统一归一化。

**代码示例：合并与映射**
```python
# 1. 合并稀有或特定头衔
title_mapping = {
    "Mr": "Mr",
    "Miss": "Miss",
    "Mrs": "Mrs",
    "Master": "Master",  # 通常指未成年的男孩
    "Dr": "Officer",
    "Rev": "Officer",
    "Col": "Officer",
    "Major": "Officer",
    "Capt": "Officer",
    "Jonkheer": "Royalty",
    "Don": "Royalty",
    "Sir": "Royalty",
    "Lady": "Royalty",
    "the Countess": "Royalty",
    "Mlle": "Miss",  # 法语小姐，归并到Miss
    "Ms": "Miss"     # 泛指女性，归并到Miss
}

for dataset in combined_data:
    dataset['Title'] = dataset['Title'].map(title_mapping)
    # 以防万一有漏网之鱼，统一标记为 'Other'
    dataset['Title'] = dataset['Title'].fillna('Other')

# 将处理后的 Title 转换为分类变量 (Category) 或进行 One-Hot 编码
train = pd.get_dummies(train, columns=['Title'], prefix='Title')
test = pd.get_dummies(test, columns=['Title'], prefix='Title')
```
*(注：做完 One-Hot 编码后，记得对齐训练集和测试集的列，防止维度不一致报错)*

### 第三步：挖掘 Name 字段的其他隐藏价值
除了头衔，Name字段还能衍生出两个有趣的特征：

#### 1. 名字长度 (Name Length)
这听起来有点玄学，但在当时，名字越长，往往代表其家族越显赫（比如带着各种中间名）。
```python
for dataset in combined_data:
    dataset['Name_Length'] = dataset['Name'].apply(len)
```
你可以后续对长度进行分箱（Binning），或者直接作为连续变量输入模型。

#### 2. 同姓族群 (Family Group / Surname Frequency)
通过提取姓氏（逗号前的部分），你可以统计船上同名的人数。这能帮你找出那些庞大的家族群体（比如著名的“阿斯特家族”或“斯特劳斯家族”）。
```python
for dataset in combined_data:
    dataset['Surname'] = dataset['Name'].apply(lambda x: x.split(',')[0])

# 统计每个姓氏在训练集中的人数
train['Family_Group'] = train['Surname'].apply(lambda x: train['Surname'].value_counts()[x])
# 将 Family_Group 分为 单独、小家庭、大家庭 等类别
```

### 💡 总结与实战建议
在泰坦尼克号这个特定的数据集中，**“Title”绝对是Name字段贡献最大的子特征**。它本质上是一个强力的“社会地位与身份标识”。

如果你是新手，建议**第一步先把 Title 提取出来做 One-Hot 编码**，通常能让你的模型分数（Accuracy/Leaderboard排名）有一个小幅但稳定的提升。当你把这套逻辑跑通后，再去尝试加入“名字长度”或“姓氏频率”等进阶特征，看看是否能进一步打破 score plateau（分数瓶颈）。祝你上分顺利！
